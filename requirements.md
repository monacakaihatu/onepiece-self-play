# ワンピースカード デッキ管理アプリ 要件定義書

> PunkDB（個人開発のファンアプリ）のデッキ管理機能をローカルで再現するプロジェクト。
> 本書はフェーズ1（デッキ管理）のスコープを定義する。一人回し機能はフェーズ2。

---

## 1. プロジェクト概要

ワンピースカードゲームのデッキを管理するローカルWebアプリを開発する。外部サービスへのアカウント登録は不要で、自分のPC上で完結して動作する。カードデータは非公式の英語APIから取得し、カード画像は日本語の公式サイトを参照するハイブリッド構成を取る。

### ゴール
- 日本語の画像でデッキを組める
- 起動・検索・スクロールが速い（ロード待ちのストレスがない）
- ローカル完結で動く

### 非ゴール（フェーズ1では作らない）
- 一人回し機能（手札・山札・トラッシュ・ライフ管理など）
- デッキのエクスポート / インポート
- 対戦記録・勝率表示
- 8パックシミュレーター

---

## 2. 技術スタック

| レイヤー | 採用技術 | 理由 |
|----------|----------|------|
| フロントエンド | React + Vite | 高速な開発体験、仮想スクロール等のライブラリが揃っている |
| バックエンド | FastAPI (Python) | ローカルAPIを軽量に立てられる、データ取得スクリプトと相性が良い |
| データベース | SQLite | ローカル完結、セットアップ不要、インデックスで高速検索 |
| 画像 | 公式JPサイト参照 + ローカルキャッシュ | 日本語画像を規約リスク低めで利用 |

---

## 3. データ取得方針（ハイブリッド構成）

### カードメタデータ
- 初回起動時に英語API（optcgapi.com 等）から全カードを一括取得し、SQLiteに保存する
- 2回目以降はローカルDBから読むため高速
- 新セット追加時のみ差分更新を実行する

### カード画像
- 公式日本語サイト（onepiece-cardgame.com）の画像URLを **カード番号から組み立てて** 参照する
  - 画像IDはカード番号と一致する（例: `OP01-001` → 画像ID `OP01-001`）
  - 別イラスト（パラレル）は `OP01-001_p1` のようにサフィックスが付く
- 一度表示した画像はローカルにキャッシュし、2回目以降は即時表示する
- ネット接続は画像の初回取得時のみ必須（メタデータはオフラインで動く）

## 4. 機能要件（フェーズ1）

### 4.1 カード閲覧・検索
- カード一覧表示（仮想スクロールで大量カードでも軽快に表示）
- 検索: カード名・カード番号での絞り込み
- フィルタ: 色 / コスト / 種別（リーダー・キャラ・イベント・ステージ）/ セット
- カード詳細表示（画像・テキスト・各属性）

### 4.2 デッキ管理
- デッキの新規作成・編集・削除
- **リーダーカードの選択は必須**（リーダー未設定ではデッキを保存できない）
- デッキ名の設定
- デッキへのカード追加・削除・枚数変更
- デッキ一覧表示 / デッキ内カード一覧表示

### 4.3 デッキ構築ルールのバリデーション（ルール通り厳密に）
- デッキ枚数: リーダー1枚 + メインデッキ50枚ちょうど
- 同名カード: 最大4枚まで
- **色制限: リーダーの色に含まれない色のカードは入れられない**（厳密にチェック）
- ルール違反時はデッキ保存をブロックし、違反内容を明示する

---

## 5. 非機能要件

### パフォーマンス（最重要）
ロード待ちのストレスをなくすため、以下を実装する。

- **DBインデックス**: カード番号・色・コストにインデックスを貼り検索を高速化
- **仮想スクロール**: カード一覧は画面に見えている分だけDOMに描画する
- **画像のlazy load**: スクロールで可視範囲に入ったタイミングで画像をロード
- **画像キャッシュ**: 一度取得した画像はローカル保存し再取得しない
- **メタデータのローカル保持**: 初回以降はAPIアクセスなしで起動

### その他
- ローカル単一ユーザー利用を前提（認証なし）
- データは全てローカルに保存

## 6. データモデル（案）

### cards（カードマスタ）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT (PK) | カード番号（例: OP01-001） |
| name | TEXT | カード名 |
| cost | INTEGER | コスト |
| color | TEXT | 色（複数色はカンマ区切り等） |
| category | TEXT | 種別（leader/character/event/stage） |
| power | INTEGER | パワー |
| counter | INTEGER | カウンター値 |
| effect_text | TEXT | 効果テキスト |
| image_url | TEXT | 日本語画像のURL |
| set_code | TEXT | セット（OP01等） |

### decks（デッキ）
| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER (PK) | デッキID |
| name | TEXT | デッキ名 |
| leader_id | TEXT (FK→cards.id) | リーダーカード（必須） |
| created_at | DATETIME | 作成日時 |
| updated_at | DATETIME | 更新日時 |

### deck_cards（デッキ-カード中間テーブル）
| カラム | 型 | 説明 |
|--------|-----|------|
| deck_id | INTEGER (FK→decks.id) | デッキID |
| card_id | TEXT (FK→cards.id) | カードID |
| quantity | INTEGER | 枚数（1〜4） |

> インデックス: `cards(id)`, `cards(color)`, `cards(cost)`, `deck_cards(deck_id)`

## 8. 留意事項・リスク

- 画像は公式サイト参照のため、URL構造が変更されると表示できなくなる可能性がある
- 英語APIは個人運営のため、停止・仕様変更のリスクがある（初回取得後はローカルDBで動くため影響は限定的）
- カード画像・データの著作権はバンダイ／集英社／尾田栄一郎氏等に帰属する。個人利用・ローカル限定の範囲で利用すること
- 日本語のカードテキストは英語APIに含まれないため、テキストは英語のまま or 別途日本語ソースの検討が必要（要フェーズ1内で判断）
